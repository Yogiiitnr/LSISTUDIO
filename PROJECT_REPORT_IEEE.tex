\begin{abstract}
This project presents LSI Acoustic Studio PRO, a comprehensive web-based digital signal processing (DSP) platform built with Streamlit, NumPy, and SciPy. The application provides real-time audio processing, convolution simulation with 8 world venues, 10 music filters, IIR filter design with 4 filter types, and advanced audio effects. The platform features a dark professional UI with intuitive controls for both beginners and DSP professionals. All functionality is verified with zero runtime errors and deployed on Streamlit Community Cloud for instant access.
\end{abstract}

\section{Introduction}
Digital signal processing is fundamental to modern audio applications. This project implements a complete DSP toolkit that allows users to:
\begin{itemize}
    \item Process audio from multiple sources (microphone, files, generated signals)
    \item Simulate acoustic spaces using impulse response convolution
    \item Apply pre-designed music filters
    \item Design custom IIR filters with interactive visualization
    \item Perform frequency domain analysis
    \item Apply real-time audio effects
\end{itemize}

    \item \textbf{Visualization}: Matplotlib 3.8+
    \item \textbf{Audio I/O}: soundfile 0.12+

\subsection{Module Organization}

    \item \textbf{Filter Library}: 10 pre-designed music filters
    \item \textbf{IIR Designer}: 4 filter types, 4 modes, magnitude/phase visualization
    \item \textbf{Analysis Tools}: Spectrogram, Power Spectral Density
    \item \textbf{Effects Processor}: Vibrato, Tremolo, Distortion, Echo
\end{enumerate}

\section{Technical Implementation}

\subsection{Audio Processing Pipeline}
All audio is normalized to [-1, 1] range at 16 kHz sample rate:
\[
x_{norm} = \frac{x - x_{min}}{x_{max} - x_{min}}
\]

\subsection{Filter Design}
Critical implementation: Nyquist normalization for scipy functions:
\[
\omega_n = \max(0.001, \min(0.999, \frac{f_c}{f_s/2}))
\]
where $f_c$ is cutoff frequency and $f_s$ is sample rate.

\subsection{Convolution Operation}
Frequency domain convolution via FFT for efficiency:
\[
y[n] = \text{IFFT}(\text{FFT}(x) \cdot \text{FFT}(h))
\]

\subsection{8 World Venues}
Acoustic simulation of iconic spaces with measured impulse responses:
\begin{itemize}
    \item Taj Mahal (5.5s decay)
    \item Sydney Opera House (3.5s)
    \item Pantheon Rome (8.0s)

    \item Church Santo Domingo (6.5s)
    \item Bamboo Grove (1.5s)
    \item Colosseum (7.2s)

\subsection{10 Music Filters}
Pre-designed EQ profiles for common use cases:
\begin{itemize}
    \item Vocal Booster, Bass Enhancer, Treble Sparkle
    \item Lo-Fi Warmth, Presence Peak, Jazz Smooth
    \item Electronic Cut, Ambient Soft, Metal Sharp
    \item Vocal Intimate
\end{itemize}

\subsection{IIR Filter Types}
\begin{enumerate}
    \item Butterworth (maximally flat passband)
    \item Chebyshev I (ripple in passband)
    \item Chebyshev II (ripple in stopband)
    \item Elliptic (ripple in both)
\end{enumerate}

\section{Critical Implementation Details}

\subsection{numpy Array Type Checking}
\textbf{Correct}: \texttt{if sos is None:} (works with numpy arrays)
\textbf{Wrong}: \texttt{if sos:} (causes ``truth value ambiguous'' error)

\subsection{Matplotlib Configuration}
Use \texttt{constrained\_layout=True} instead of \texttt{tight\_layout()} for proper rendering.

\subsection{Session State Management}
\begin{itemize}
    \item \texttt{st.session\_state.audio}: np.ndarray or None
    \item \texttt{st.session\_state.sr}: int (16000 Hz)
\end{itemize}
